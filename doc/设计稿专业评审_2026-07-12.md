## 1. 整体评估

### 1.1 视觉方向

- **品牌色与调性**：深色 slate（`#0A0E17`）+ 霓虹网球绿（`#D4F820`）的配色已经形成较强记忆点，符合“运动、竞技、数字名片”的调性。
- **信息架构**：7 个页面分工清晰：名片 / 战绩 / 排行（Tab 浏览）+ 发起比赛 / 准备室 / 比分录入 / 能力测评（任务流程）。流程顺畅。
- **设计语言**：卡片 + 圆角 + 内阴影的容器语言一致，主要数据（ELO、比分、排名）都使用了大字号 Oswald 强调，层级基本清晰。

### 1.2 总体问题

- **P0：「发起比赛」常驻按钮在 Tab 页不可见**。这是功能入口级的严重问题，会直接导致用户无法发现核心操作。
- **P1：排行榜 Top3 奖杯台视觉失衡**，1 号位卡片比例过大、数字溢出。
- **P1：战绩页双打卡片信息拥挤**，在移动端可读性差。
- **P2：部分占位图、弱对比文字、进度条等细节需要打磨。**

## 4. 优先级修复清单（供工程排期）

2. **修复排行榜 Top3 奖杯台比例**，三卡片等宽（1 号位可稍宽但不超过 12px），数字不溢出卡片。
3. **修复战绩大厅双打卡片布局**，避免长队名与比分在同一行挤压。

### P1 — 建议本期修复（强影响体验）

6. 提升排行榜副标题与标题的层级对比，避免与小程序胶囊按钮重叠。

### P2 — 细节打磨

## 5. 设计系统建议

- **间距**：建议建立 4pt/8pt/12pt/16pt/24pt/32pt 的栅格，避免页面中的硬编码间距。
- **卡片**：统一圆角 16px、描边 1px @ 6% 白色、内阴影仅用于强调卡片（如积分卡）。
- **颜色**：保持 1 个强调色（网球绿），灰阶使用 `#6B7286` → `#A0A8B8` → `#D1D5DB` 的阶梯，确保信息层级。
- **字体**：Oswald 用于大数字和标题，DM Sans 用于正文和标签，字号阶梯：12/14/16/24/32/48。
- **阴影**：减少外发光，改用基于背景色的微弱投影（如 `rgba(212,248,32,0.15)` 或深灰投影）。
