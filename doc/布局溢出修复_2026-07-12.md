## 2. 根因（系统性，非单页问题）

5 个自定义导航流程页的滚动区均使用以下脆弱写法：

```css
/* 旧：scroll-view 高度 = 100vh 减去 头/底 预留，再减 safe-area */
height: calc(100vh - <headerReserve>px - <footerReserve>px - env(safe-area-inset-bottom));
```

问题点：**`<footerReserve>` 是硬编码的估算值，且未包含 footer 自身的 `env(safe-area-inset-bottom)` 内边距**。

- 在非刘海设备（safe-area = 0）：footer 实际高度 < 预留 → 仅留少量底部空白，不溢出。
- 在刘海屏设备（safe-area > 0，如 iPhone 模拟器 ≈ 34px）：footer 实际高度 = `原估算 + 34px` > 预留 → **scroll-view 被算得过高 → 固定 footer 被推过屏幕底边 → 内容/按钮被裁切（"超出屏幕边界"）**。

各页溢出量估算（以 safe-area = 34px 计）：

| 页面 | footerReserve | footer 实际(含safe) | 溢出 |
| --- | --- | --- | --- |
| assessment | 88px | ~118px | **~30px** |
| create | 100px | ~118px | **~18px** |
| confirm | 200px | ~216px | **~16px** |
| room | 130px | ~144px | **~14px** |
| score-entry | 380px | ~368px | 临界（无安全余量） |

---

## 3. 修复方案

根容器**本就已具备**正确的 flex 结构（`display:flex; flex-direction:column; height:100vh; overflow:hidden`，且 header / footer 均为 `flex-shrink:0`）。问题仅在于 scroll-view 用了"手动估算高度"而非"让 flex 实测剩余空间"。

**改法**：将 scroll-view 的 `calc(100vh - …)` 高度替换为：

```js
const scrollStyle = computed(() => ({
  flexGrow: '1',
  flexShrink: '1',
  flexBasis: '0%',
  minHeight: '0',   // 关键：允许 flex 子项收缩到内容高度以下，从而内部可滚动
  width: '100%',
}));
```

等价 CSS：`flex: 1 1 0%; min-height: 0;`。这样 scroll-view 高度 = `100vh − 真实 header 高 − 真实 footer 高`，**无论 safe-area 取值多少都精确贴合，彻底免疫溢出**。footer 自身的 `paddingBottom: calc(12px + env(safe-area-inset-bottom))` 保持不变，safe-area 由 footer 层独立承担。

5 个页面（`create` / `room` / `score-entry` 的 `topStyle` / `confirm` / `assessment`）统一采用此改法，模板与样式结构零改动，仅替换 scroll-view 的高度计算。

## 4. 逐页 Review 详情

### 4.1 已修复的 5 个流程页
- 根容器 `flex` 列布局 + header/footer `flex-shrink:0` 已就绪，仅需 scroll-view 改 `flex:1; min-height:0`。
- 修复后 scroll-view 精确填充 header 与 footer 之间，footer 不再被推过屏幕底边，最后一条内容距 footer 由页面内 `12~24px` 占位条保证呼吸空间。

### 4.2 TabBar 三页（无需改）
- `index` / `history` / `leaderboard` 采用"**固定 TabBar（`components/tabbar`，wrapper 已含 `env(safe-area-inset-bottom)` padding）+ scroll-view 预留 `TABBAR_HEIGHT`(52px)**"模式。
- 预留已含 safe-area 且偏保守（header 预留 > 实际），scroll 内容末尾有 `h-8` / `pb-8` 占位，TabBar 不遮挡内容，**无溢出/重叠**。

### 4.3 组件重叠检查（无问题）
- `tabbar`：`position:fixed; bottom:0; z-index:100`，高度 = 50px 内栏 + 1px 边框 + safe-area，与预留一致。
- `create-fab`：`position:fixed; right:16px; z-index:999; bottom: calc(TABBAR_HEIGHT + env(safe-area-inset-bottom) + 16px)` → 始终悬浮于 TabBar **之上** 16px，无重叠；且仅挂载于 3 个 TabBar 页，流程页不挂载。
- 驳回弹窗（`confirm.vue`）：`fixed inset-0 z-50`，层级低于 FAB 但 confirm 页无 FAB，无冲突。
- 横向溢出：微信 `box-sizing` 默认 `border-box`，`width:100% + px-6` 内边距不撑破屏幕，无横向越界。
