# 微信端 `Error: timeout` 修复报告（2026-07-12）

## 问题现象
微信开发者工具控制台报错：
```
Error: timeout
    at Function.<anonymous> (WAServiceMainContext.js?t=wechat&v=3.15.1:1)
    ...
(env: Windows, mp, 2.01.2510290; lib: 3.15.1)
```

## 根因
这不是设计/样式问题，而是**后端连通性问题**：

- 前端所有 API 请求（`/api/users/me`、`/api/login`、`/api/match/list`、`/api/leaderboard` 等）统一发往 `VITE_BACKEND_URL = http://localhost:3000`（见 `client/.env`）。
- 微信小程序运行时**没有任何服务监听 `localhost:3000`** → `wx.request` 一直挂起，直到基础库抛出 `Error: timeout`。
- `pages/index/index.vue` 在 `onMounted` 时若检测到本地存有 `auth_token`，会调用 `/api/users/me` —— 这正是启动即超时的来源。

> 此前的「与设计稿完全不同」是**陈旧构建**导致（已修复）；本次的 `Error: timeout` 是**无后端**导致，二者根因不同。

## 修复方案
新增**前端离线预览模式（Mock）**，让小程序无需后端即可在开发者工具中完整渲染设计稿与交互；同时加固请求层。

### 1. 新增 `client/src/utils/mock.ts`
内置 Mock 数据（用户名片、战绩列表、排行榜、登录、测评提交），`getMockResponse(url, method, data)` 按路径返回对应数据。未覆盖的接口兜底返回 `{ success: true, data: {} }`，**任何情况下都不抛 timeout**。

### 2. 加固 `client/src/utils/request.ts`
- 导出 `USE_MOCK = import.meta.env.VITE_USE_MOCK === 'true'`
- Mock 模式下 `request()` 直接 `Promise.resolve(getMockResponse(...))`，**完全不走网络**
- 真实模式下 `uni.request` 增加 `timeout: 10000`，避免无限挂起

### 3. `pages/index/index.vue` 登录流程兼容
`handleWechatLogin` 在 Mock 模式下跳过真实 `uni.login`（开发者工具无微信账号时会失败），改用 `mock-code` 兜底，登录后直接拿到 Mock 用户并渲染「我的名片」。

### 4. `client/.env`
```
VITE_USE_MOCK=true          # 默认开启离线预览
VITE_BACKEND_URL=http://localhost:3000
```
注释中说明联调真实后端时的切换方式。

## 验证结果
| 步骤 | 结果 |
|------|------|
| `build:mp-weixin` | `EXIT=0`，零错误，`DONE Build complete.` |
| `dev:mp-weixin` | `DONE Build complete.`（产物刷新至 08:47） |
| Mock 已打入产物 | `dist/build/mp-weixin/utils/mock.js` 含 Mock 数据；`utils/request.js` 引用 `getMockResponse` ✓ |

## 开发者工具中如何确认
重新「编译」或重新「导入」`client/dist/dev/mp-weixin`（或 `dist/build/mp-weixin`）。现在：
- 首页显示登录页（无需后端）；点「微信一键登录」→ 直接进入 Mock 的「我的名片」页（含雷达图、战绩摘要）。
- 切到「战绩」Tab → 看到 Mock 比赛列表（含待确认黄条）。
- 切到「排行」Tab → 看到 Top3 奖牌台 + 我的排名。
- 不再出现 `Error: timeout`。

## 如何联调真实后端
1. **本地**：在项目根目录运行 `npx vercel dev`（默认 `:3000`，会 serve `api/` 下所有 Serverless 函数）。`manifest.json` 中 `mp-weixin.setting.urlCheck: false` 已关闭域名校验，小程序可直连 `localhost:3000`。
2. **线上（部署后）**：将 `.env` 的 `VITE_BACKEND_URL` 改为已部署的 Vercel HTTPS 地址，把该域名加入微信公众平台「request 合法域名」白名单，并将 `VITE_USE_MOCK` 置为 `false`。

## 小结
`Error: timeout` 的本质是「小程序在喊后端，但没人应答」。现已用前端 Mock 让设计稿可离线完整预览；需要真实数据时，按上述任一方式把后端接通即可。
