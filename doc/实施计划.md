| 层级 | 技术 |

#### 2. 比赛创建 API（预计 1 天）
**新增文件**：`api/match/create.ts`

**接口设计**：
```
POST /api/match/create
Authorization: Bearer <token>
Body: {
  match_type: 1 | 2,  // 1=单打 2=双打
  invitee_openid?: string  // 可选：指定邀请对象
}

Response: {
  success: true,
  data: {
    match_id: "uuid",
    match_type: 1,
    status: "waiting",
    share_ticket: "encoded_string"  // 用于分享给对手的凭证
  }
}
```

**业务逻辑**：
1. 验证 Token，获取当前用户
2. 创建 `matches` 记录（status=0 等待中）
3. 创建 `match_participants` 记录（creator 的快照积分）
4. 返回 `match_id` 和 `share_ticket`

#### 3. 比分录入页（预计 2 天）
**新增页面**：`client/src/pages/match/score.vue`

**功能**：
- 显示双方球员信息（头像 + 昵称 + 开赛快照积分）
- 比分盘：双 6 格布局，tap 切换格点亮状态
- 支持 1-6 的局比分录入
- 提交后调用 `POST /api/match/:id/score`

**接口**：`POST /api/match/[id]/score`
```
Body: {
  score_a: number,  // A队局分
  score_b: number   // B队局分
}
```

#### 4. 比赛确认/争议页（预计 1 天）
**新增页面**：`client/src/pages/match/confirm.vue`

**功能**：
- 展示对方提交的比分
- 确认✅ / 争议❌ 两个按钮
- 超时 12 小时自动生效逻辑（后端 `timeout_at` 字段）
