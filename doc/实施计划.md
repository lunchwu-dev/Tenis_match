> 最后更新：2026-04-22

## 一、当前实施状态

### 1. 构建与代码健康
- 已新增根目录统一构建入口：`pnpm run build`
- 2026-04-22 本地已通过以下验证：
  - `pnpm --filter client run type-check`
  - `pnpm --filter client run build:h5`
  - `pnpm --filter api run type-check`
  - `pnpm --filter api run build`
- 当前唯一已知非阻塞构建告警：uni build 阶段存在一个 PostCSS 插件未传递 `from` 参数的 warning，不影响产物生成，但建议后续排查具体插件来源。

### 2. 已实现能力盘点

| 模块 | 状态 | 说明 |
|------|------|------|
| Monorepo 构建脚本 | ✅ 已完成 | 根目录已支持统一 type-check 与全量 build |
| 用户/测评 API | ✅ 已完成 | `login`、`evaluate`、`assessment`、`users` 目录代码已通过编译 |
| 比赛状态机 API | ✅ 已完成 | `create/join/start/submit-score/confirm/reject/timeout-settlement` 已有实现并通过编译 |
| 排行榜 API | ✅ 已完成 | Redis + DB fallback 逻辑已通过编译 |
| 首页/测评/比赛相关页面 | ⚠️ 部分完成 | 页面结构已具备，但部分数据仍为 mock 或未完成 API 联调 |
| 历史/排行榜页面 | ⚠️ 部分完成 | 当前以前端静态展示为主，尚未全部接入真实数据 |

### 3. 当前主要差距与风险
1. 首页雷达图虽已具备 Canvas 绘制框架，但仍使用 mockData，未绑定真实 `radar_data`。
2. 比赛创建页仍会生成 mock `matchId`，建房与准备室流程尚未完全切到真实后端链路。
3. 排行榜页、历史战绩页当前仍以静态 UI 为主，未完成与后端数据的完整联调。
4. `api/login.ts` 在环境变量缺失时仍保留默认 `WX_APP_SECRET`，属于发布前必须移除的安全风险。
5. `api/evaluate.ts` 保留本地 `token__` 测试路径，正式环境需统一认证行为与异常处理。
6. Vercel、Supabase、Upstash、微信小程序真环境是否已全部配置完成，当前仓库内无法直接验收，需要单独核对。

## 四、阶段验收口径

### MVP 联调通过的最低标准
- 根目录 `pnpm run build` 持续通过。
- 小程序端可从登录/测评进入首页，并展示真实积分与雷达图。
- 单打/双打至少一条比赛链路可从建房跑到确认结算。
- 排行榜与历史页展示真实用户/比赛数据，而非静态 mock。
- 默认密钥与本地测试兜底逻辑不再出现在生产链路。

### 发布前补充标准
- 超时结算与 Cron 在真环境验证通过。
- 微信开发者工具无阻塞级报错。
- 关键接口具备最小回归清单与人工验收记录。
