# 前端架构与 UI 像素级还原指南 (UniApp + Tailwind + TS)

**文档目的**：基于确定的现代化技术栈（UniApp Vue 3 + TypeScript + Tailwind CSS + weapp-tailwindcss + uCharts），将单页面的 UX 原型 (`ux_prototype.html`) 进行工程化拆解，提取全局配置、规划组件目录，并提供标准化代码范例，作为前端落地的直接执行标准。

## 1. 全局设计系统配置 (Design Tokens)

为了在小程序中 100% 还原暗黑荧光风，我们需要将原型中的写死样式抽取到 `tailwind.config.js` 中。通过 `weapp-tailwindcss`，这些类名将被完美编译为独立于微信小程序的 WXSS。

```javascript
// tailwind.config.js
module.js.exports = {
  content: ['./index.html', './src/**/*.{html,js,ts,jsx,tsx,vue}'],
  theme: {
    extend: {
      fontFamily: {
        // 小程序端需在 App.vue 中通过 @font-face 引入网络字体或 Base64
        display: ['Oswald', 'sans-serif'], 
        body: ['DM Sans', 'sans-serif'],
      },
      colors: {
        'tennis-neon': '#D4F820', // 主题荧光绿 (高亮CTA, 极高对比度)
        'dark-slate': '#0A0E17',  // 页面主层底层色 (最深的夜空黑)
        'card-bg': '#151A26',     // 卡片/模块底色 (带轻微蓝相的深灰)
        'accent-blue': '#2E68FF', // 辅助点缀蓝 (用于认证标识等)
      },
      boxShadow: {
        'neon-glow': '0 0 40px rgba(212,248,32,0.3)', // 头像/大按钮发光
        'neon-glow-sm': '0 10px 25px -5px rgba(212,248,32,0.4)', // 悬浮按钮发光
        'inner-glow': 'inset 0 0 20px rgba(212,248,32,0.05)', // 卡片内发光
      }
    }
  },
  plugins: [],
  // 必须配合 weapp-tailwindcss 的 vite plugin 使用
}
```

## 3. 标准化转换范例 (Vue 3 + TS + Tailwind)

### 范例 A：战绩大厅卡片组件 (`components/TmMatchCard.vue`)
利用 `<script setup lang="ts">` 严把数据流入，结合原子化 CSS 实现解耦。

```vue
<template>
  <!-- 动态改变边框与透明度，区分胜负状态 -->
  <view 
    class="p-4 rounded-2xl border relative overflow-hidden transition-all"
    :class="[
      isWin ? 'bg-card-bg border-gray-800' : 'bg-card-bg opacity-70 border-gray-800'
    ]"
  >
    <!-- 左侧状态指引条 -->
    <view class="absolute left-0 top-0 bottom-0 w-1" :class="isWin ? 'bg-tennis-neon' : 'bg-red-500'"></view>
    
    <!-- 头部信息 -->
    <view class="flex justify-between items-center mb-3">
      <text class="text-xs text-gray-500">{{ date }} · {{ matchType }}</text>
      <text class="font-display font-bold text-sm" :class="isWin ? 'text-tennis-neon' : 'text-red-500'">
        {{ isWin ? '+' : '' }}{{ pointChange }} 分
      </text>
    </view>

    <!-- 对阵展示 -->
    <view class="flex items-center justify-between">
      <!-- 己方 -->
      <view class="flex items-center gap-2">
        <image :src="myAvatar" class="w-8 h-8 rounded-full border border-gray-700" mode="aspectFill" />
        <text class="font-bold text-sm text-white">我方</text>
      </view>
      
      <!-- 比分 -->
      <view class="px-3 py-1 bg-gray-800 rounded font-display font-bold text-lg tracking-widest text-white">
        {{ score }}
      </view>
      
      <!-- 敌方 -->
      <view class="flex items-center gap-2 flex-row-reverse">
        <image :src="opponentAvatar" class="w-8 h-8 rounded-full border border-gray-700" mode="aspectFill" />
        <text class="text-sm text-gray-400">{{ opponentName }}</text>
      </view>
    </view>
  </view>
</template>

<script setup lang="ts">
import { computed } from 'vue';

// 强类型约束组件 Props
interface MatchCardProps {
  date: string;
  matchType: '单打' | '双打';
  pointChange: number;
  score: string;
  myAvatar: string;
  opponentName: string;
  opponentAvatar: string;
}

const props = defineProps<MatchCardProps>();

// 状态判定逻辑
const isWin = computed(() => props.pointChange > 0);
</script>
```

### 范例 B：将 uCharts 嵌入我的名片页 (`pages/index/index.vue` 节选)

在我们的原型 HTML 中，雷达图用的是 CSS `clip-path` 的视觉欺骗。在真实架构中，我们必须利用 `@qiun/ucharts` 进行动态渲染，以映射用户的真实五维数据。

```vue
<template>
  <view class="w-full flex justify-center py-4 relative">
    <view class="w-48 h-48 relative">
      <!-- uCharts 跨端原生 Canvas 图表组件 -->
      <qiun-data-charts 
        type="radar"
        :opts="radarOpts"
        :chartData="radarData"
        background="none"
      />
    </view>
  </view>
</template>

<script setup lang="ts">
import { ref } from 'vue';

// 图表配置，精准匹配暗黑霓虹风设计稿
const radarOpts = ref({
  color: ["#D4F820"], // 霓虹绿网网多边形
  padding: [5, 5, 5, 5],
  dataLabel: false,
  legend: { show: false },
  extra: {
    radar: {
      gridType: "radar",
      gridColor: "rgba(255,255,255,0.1)",   // 底层雷达蛛网颜色
      gridCount: 3,
      opacity: 0.2,                         // 填充透明度
      labelColor: "#9ca3af",                // tailwind text-gray-400
      border: true,                         // 显示多边形线条
      borderWidth: 2
    }
  }
});

// Vue 3 响应式五维数据 (待由接口注入)
const radarData = ref({
  categories: ["底线", "发球", "网前", "战术", "接发"],
  series: [
    {
      name: "当前能力",
      data: [80, 50, 40, 90, 70]
    }
  ]
});
</script>
```

## 5. 小程序多机型适配与屏幕安全区指南 (Device Adaptation & Safe Area)

作为一个全屏沉浸式的“暗黑霓虹风”应用，传统的原生白色/灰色导航栏会严重破坏视觉氛围。因此，针对微信小程序不同机型的硬件差异（刘海屏、灵动岛、底部小黑条），需要重点补充以下适配策略与资源：

### 5.1 顶部通栏与自定义导航栏 (Custom Navigation Bar)
为了让界面顶部的深色渐变能够顶到屏幕边缘（沉浸式体验），必须在 `pages.json` 中配置为自定义导航栏：
```json
{
  "globalStyle": {
    "navigationStyle": "custom",
    "navigationBarTextStyle": "white"
  }
}
```
**胶囊适配补齐**：在使用自定义导航栏后，所有的固定头部区域（如“能力定级页”的返回按钮与进度数字）都必须避开微信右上角的“胶囊按钮”。
*   **开发方案**：编写一个全局的 `TmNavBar.vue`，在组件内部调用 `uni.getSystemInfoSync().statusBarHeight` 获取状态栏高度，并利用 `uni.getMenuButtonBoundingClientRect()` 获取胶囊位置，动态计算出顶部预留的空白高度 (`padding-top`)，让页面内容安全对齐。

### 5.2 底部 Home Indicator 安全区 (Safe Area Bottom)
在长屏幕（如 iPhone X 及后续机型）底部会有横条（Home Indicator），如果你绘制的底部主操作按钮（悬浮 CTA）或底部 TabBar 不做安全区避让，会被系统的横条遮挡，导致极大概率出现误触点回桌面的情况。
*   **Tailwind 适配方案**：利用 CSS 环境变量，在我们写 Tailwind 类名时通过注入 safe-area：
    *   在原生的外层容器加入：`pb-[env(safe-area-inset-bottom)]`。
    *   我们在 UX 稿中的 `safe-area-bottom` 类需要在全局样式中真实实现为：`padding-bottom: calc(20px + env(safe-area-inset-bottom));`。

### 5.3 响应式弹性布局 (rpx vs % 分配)
*   **纵向自适应**：像 iPhone SE 这类小屏（高度不足 700px）很容易装不下我们的一屏问卷卡片。因此所有主界面容器必须使用 `flex-1 overflow-y-auto` 或 `ScrollView` 进行包裹，避免内容截断。
*   **尺寸单位选择**：
    *   文字、间距、常规卡片宽度，直接使用 Tailwind 默认的 `rem/px` 等同比单位，保持在各屏幕上物理大小一致，防止在大屏手机上显得异常硕大。
    *   **五维雷达图、固定比例的雷达蛛网**，建议在传给 uCharts 宽高时使用 `rpx` (如 `w-[400rpx] h-[400rpx]`)，确保其在各种宽度下严格保持视觉中的对等占比中心距。

### 5.4 微信字体缩放特性 (Font Scaling Control)
如果用户在微信里设置了超大字体（老人机模式），会把我们的精心排版撑破甚至发生严重的重叠。对于这种高度风格化、类似游戏 UX 界面的应用，必须锁死系统字体大小影响：
*   **开发方案**：小程序提供了全局控制能力。在 `app.js`（或 `App.vue` 生命周期）中通过 `uni.setSystemInfo({ fontSizeSetting: 16 })`，或者在页面配置中开启忽略系统字体。
