## 🛠 2. 各角色的工作切入流 (Role-based Workflows)

为了最高效地利用这套文档，建议不同的角色按照以下路径进行阅读与落地：

### 👨‍💼产品经理 (PM)
1. `00` 和 `01`：重温项目愿景和边界，防止需求蔓延（Feature Creep）。
2. `02`、`03`、`05`：检查问卷和积分算法设定是否满足网球运动规则、数值是否自洽。
3. `progress.md`：每周更新迭代进度。

### 🎨UI/UX 设计师 (Designer)
1. `ux_prototype.html`：**直接在浏览器打开该文件**，这是您后续所有高保真视觉稿的组件基础与交互样式参考基准。
2. `04` 和 `06`：这是您的核心输入需求，根据页面流转图和元素要求，进一步打磨细化设计切图和动画说明。

### 👨‍💻前端工程师 (Frontend / FE)
1. `ux_prototype.html`：提取此文件中的 TailwindCSS 色值与类名直接复用。
2. `06`：按照流转图（建房、状态同步、超时）与交互说明挂载事件。
3. `02`：根据这个文档将 23 题定级问卷静态写成配置文件（JSON）。
4. `07`：了解整体服务通信方案，准备接入 Vercel 提供的 API 接口。

### 👨‍💻后端工程师 (Backend / BE) / 架构师
1. `07_技术架构与部署方案设计.md`：这是您的**工作起点**。按照文中的表结构和 Serverless 部署要求搭建后台并配置数据库、Redis。
2. `03_比赛结算与Elo积分引擎.md`：核心工作量。必须照着公式与防作弊衰减示例实现核心 `EloService` 类。
3. `06`：处理各种异常流（如分布式锁防重复请求）。
4. `09_环境密钥与配置(勿传GIT).md`：您部署上云的认证凭据（用后即闭，勿删防漏）。

### 🕵️‍♂️测试工程师 (QA)
1. `06_页面交互与流程设计.md` 里的【异常情况与防御性设计 (Exception Handling)】。
2. `08_项目执行与实施计划.md` 里的【测试验收边界与标准（包含12H超时不点过审验证）】。以此撰写测试用例。
