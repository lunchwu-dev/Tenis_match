# 三项体验改造报告（2026-07-12）

针对三个微信小程序体验问题完成重新设计、编码与构建验证。

## 1. 底部 TabBar 高度降低（提升屏效）

**问题**：原 TabBar 内层高度 `80px` + 顶部内边距 `12px` ≈ 93px，占用过多屏幕纵向空间；且各页 scrollStyle 硬编码 `80px`，难以统一维护。

**方案**：
- 新增全局常量 `client/src/utils/layout.ts`：`export const TABBAR_HEIGHT = 52`（= 组件内层 50px + 1px 边框），作为各页 scroll 计算的唯一来源。
- 重构 `components/tabbar/index.vue`：内层高度 `80→50px`、图标 `24→20px`、标签 `10/11px`、去掉多余顶部留白。整体降低约 45%。

**约定**：后续若调整 TabBar 高度，必须同步改 `layout.ts`，否则各页可滚动区域会与 TabBar 错位。

## 2. 常驻「发起比赛」悬浮按钮

**问题**：原「发起比赛」入口埋在 `index` 页 scroll-view 内部，随页面滚动移出视口，无法"每页立即发起"。

**方案**：
- 新增 `components/create-fab/index.vue`：
  - `position: fixed; right: 16px;`，`bottom: calc(TABBAR_HEIGHT + env(safe-area-inset-bottom) + 16px)`
  - 霓虹绿胶囊「＋ 发起比赛」，`box-shadow` 霓虹辉光，`z-index: 999` 悬浮于 TabBar 之上
  - 点击 `uni.navigateTo({ url: '/pages/match/create' })`，`hover-class` 按压缩放
  - 不随页面滚动，始终可见、可点
- 接入三 Tab 页（index / history / leaderboard）模板；`index` 同时移除 scroll 内冗余按钮及 `startMatch`。

**范围说明**：FAB 挂载于三大浏览页。比赛流程页（create / confirm / score-entry）与测评页是带固定底部操作栏的临时流程，挂 FAB 会与操作栏重叠，故按工程判断不挂；如需字面"每页"可补挂。

## 3. 排行页重构

**问题**：
1. 标题右侧「上海徐汇 · Top 100」与小程序右上角胶囊按钮（••• ⊙）同高重叠；
2. Top3 奖牌台卡片用 `flex-1` 但缺 `min-w-0`，长昵称撑破右边界；
3. 含 3 个 Tab（同城/好友/全国），需求收敛为同城+好友两榜、同城暂未实现。

**方案**：
- Tab 收敛为「同城 / 好友」两榜，移除「全国」；同城仅占位提示「同城榜单功能开发中」，默认选中「好友」展示真实数据。
- **重叠修复**：位置信息从标题右侧移至标题下方左对齐副标题「上海徐汇 · 好友积分榜」，彻底避让胶囊按钮。
- **溢出修复**：
  - Top3 卡片：`flex-1 min-w-0` + 昵称 `truncate`（容器内可收缩，长昵称省略号截断）；
  - 列表行：左侧信息组 `min-w-0 flex-1` + 昵称 `truncate`，分数 `flex-shrink-0` 固定右对齐。
- scrollStyle 改用 `TABBAR_HEIGHT`，header 高度估算同步更新。
