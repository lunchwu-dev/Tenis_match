**对比方法**：由于当前模型不可直接读取截图（`.webp`），改为通过 ARDot `batch_read` 读取设计稿节点 JSON（布局/尺寸/文案），与编译产物 `dist/build/mp-weixin/**` 的 `.wxml`/`.wxss` 做结构级比对。

## 2. 核心结构偏差（最关键）

设计稿中 **「发起比赛(3:311)」与「赛前准备室(3:375)」是两个独立页面**；但修复前前端只有 6 个页面，且 `pages/match/create.vue` 实为"准备室"内容（含二维码、席位、开始按钮），**缺失独立的发起比赛表单页**。

→ 修复方案：将 `create` 与 `room` 拆分为两个页面，各自对齐设计稿对应屏。

---

## 3. 详细修复清单

### 3.1 拆分 Create / Room 两页

**`client/src/pages.json`**
- 注册新页 `pages/match/room`（style: `navigationStyle:custom`、`navigationBarTitleText:"比赛准备室"`）。

**`client/src/pages/match/create.vue`（重写为设计稿版「发起比赛」）**
- 顶部导航：`‹ 发起比赛`（左返回 / 居中标题）。
- 模式选择卡：单打 1v1 / 双打 2v2 两卡，选中态 `mode-selected`（绿框 `border:#D4F820` + 绿色辉光阴影），未选中 `mode-normal`（card-bg 灰卡）。
- 场地卡片：📍 图标 + 名称 + 球场 + `›`，`pickVenue()` 用 `uni.showActionSheet` 选场地。
- 机制提醒卡：`tip-card` 蓝边（`border: rgba(46,104,255,0.3)`），文案"12 小时内对方未确认将自动结算"。
- 底部固定霓虹绿「创建比赛」按钮；`createMatch()` 成功后 `navigateTo /pages/match/room?matchId=…`。

**`client/src/pages/match/room.vue`（新建，设计稿版「赛前准备室」）**
- 状态横幅 `status-banner`：绿点 + `等待对手加入 · x/y` / `全员就绪，可开启比赛`。
- 比赛席位 `slots-card`：主队 filled（绿边）/ empty（虚线 + ＋）、客队 filled / empty；filled 显示头像首字、昵称、房主标记、`ELO · NTRP`；`emptySlotA/B` 计算空位数。
- 邀请二维码 `qr-card`：白色圆角卡 + 黑底 `qr-box` + `qr-grid` 21×21 点阵（`qrFlat`，由 `buildQr(code)` 基于邀请码确定性生成，含三角定位块）+ 邀请码提示。
  - ⚠️ **微信端亚像素修复**：网格 `width:147px; height:147px`，`qr-cell { width:7px; height:7px }`（21×7=147）。原 140px/6.67px 易因亚像素在小程序端折行错位。
- 复制邀请码行 + `开始比赛` 按钮（`confirmStart` → `POST /api/match/start`，满员才可点）。
- `loadMatch`：`GET /api/match/:id` 加载 creator + teamA + teamB，判定房主。

### 3.2 战绩大厅卡片（history/index.vue）

原卡片为单行拥挤布局，改为设计稿的**水平两行**：
- 行1：左侧 `日期 · 类型`，右侧 `▲/▼ ±ELO`（胜绿 / 负红）。
- 行2：左 = 我方头像首字 + `胜/负` + 我方组合名（truncate）；中 = 居中大比分 `scoreA : scoreB`（font-display 2xl）；右 = 对手组合名 + `负/胜` + 对手头像首字。
- 新增 `myTeamInitial / myTeamNames / oppTeamNames`（依据 `currentUserTeam` 取 A/B 队，支持双打组合名）；移除废弃的 `getSelfName / getSelfAvatar / getOpponentAvatar`。
- 胜负色：`.match-win` 绿边（rgba(212,248,32,0.3)） / `.match-lose` 灰边 + 轻微降透明 / `.match-pending` 灰边；卡片加 `hover-class` 按压态。

### 3.3 提交比分（score-entry.vue）

比分区由分散布局改为设计稿的 **`score-board` 卡片**：
- 两行 `team-row`（左 `team-info` 头像+队名，右 `score-box` 比分框）。
- 中间 `vs-line`（`VS` 居中，原为过淡散排）。
- `score-box` 三态：`-normal`（灰底+下边框）、`-active`（绿框+辉光，A 队）、`-active-blue`（蓝框+辉光，B 队）。
- 保留自定义数字键盘（1-9 / 0 / 换边 / ⌫）与提交按钮逻辑不变。

### 3.4 对齐细节

- **assessment/index.vue**：返回图标 `‹` → `✕`（对齐设计稿关闭 X 语义）。
- **index/index.vue**：综合积分数字 `text-white` → `text-tennis-neon`（设计稿积分霓虹绿，第 87 行 `scoreInt`）。

### 3.5 Mock 支撑（utils/mock.ts）

- `GET /api/match/:id` 返回完整比赛信息（`creator` + `teamA` + `teamB`），`teamB` 置空（单打仅主队占位），支撑 `room.vue` 离线预览。

## 6. 已知限制 / 待办

1. **二维码为装饰性占位**：`buildQr()` 是基于邀请码确定性生成的伪点阵，非真实可识别 wxacode / 二维码。生产环境需替换为微信 `getwxacode` 或真实二维码库。
2. **前端认证**：`create.vue` / `room.vue` 的 `userId` 取自 `storage.user_info`，真实后端联调需开启 `VITE_USE_MOCK=false` 并确保 `auth_token` 注入（见 `2026-07-12` 上午日志的联调文档）。
3. **场地选择**：`pickVenue()` 暂用示例 ActionSheet，未接入真实场馆/地图选择器。
4. **设计稿其余 4 屏**（我的名片 / 战绩大厅整体 / 排行榜 / 能力测评）已在前序任务对齐，本轮仅局部微调（积分色、关闭图标）。如需逐像素终验，建议导出最新设计并人工对照截图。
