# 微信小程序端渲染「与设计稿不一致」诊断报告

_生成时间：2026-07-11_

## 一、结论（先看这里）

**代码实现本身没有问题，编译产物与设计稿在结构、配色、布局上是一致的。**
你看到的「完全不一样」，几乎可以锁定在以下三类原因之一，而非前端逻辑错误：

1. **导入目录错误**（最高概率）—— 微信开发者工具必须导入 `client/dist/build/mp-weixin` 这个**具体目录**，导入上层目录会导致白屏/默认样式。
2. **用了 dev 产物但工具指向不对** —— `dist` 下有两个产物目录（`dev` 与 `build`）。
3. **微信平台固有限制**（字体/毛玻璃/按压反馈）叠加，观感差异被放大。

## 三、最可能的根因与解决方案

### 原因 1（最高概率）：导入目录不对

微信开发者工具导入项目时，**目录必须精确到 `mp-weixin` 那一层**：

```
✅ 正确：f:\Code-Base\Tenis_match\client\dist\build\mp-weixin
❌ 错误：f:\Code-Base\Tenis_match\client            （UniApp 源码，无 app.json）
❌ 错误：f:\Code-Base\Tenis_match\client\dist        （无 app.json）
❌ 错误：f:\Code-Base\Tenis_match                    （根目录，无 app.json）
❌ 错误：f:\Code-Base\Tenis_match\client\src         （.vue 源码，微信工具无法编译）
```

**导入错误时的表现**：微信工具找不到 `app.json` → 报错 / 显示空白 / 或渲染成微信默认空模板（白底 + 默认 tabBar）—— 这正是「完全不一样」的典型症状。

**正确步骤**：
1. 微信开发者工具 → 导入项目
2. 项目目录选择 `...\client\dist\build\mp-weixin`
3. AppID 可用「测试号」或你自己的小程序 AppID
4. 导入后等待编译完成

### 原因 2：dev 与 build 产物混淆

- 如果你跑 `pnpm run dev:mp-weixin` → 产物在 `dist/dev/mp-weixin`
- 如果你跑 `pnpm run build:mp-weixin` → 产物在 `dist/build/mp-weixin`

两个目录**都能运行**，但请确保工具导入的是**带 `mp-weixin` 的那一层**，而不是 `dist` 或 `client`。

> 推荐用 `build` 产物导入微信工具（更干净、更接近发布态）。开发联调用 `dev` 也行。

### 原因 3：后端服务未运行，数据为空

前端大量内容（排行榜、战绩、用户积分）依赖后端 API。如果本地没有运行后端（`api/` 下的 Serverless 函数需部署到 Vercel 或本地模拟），页面会显示空状态（如「暂无数据」），但**深色背景、tabBar、标题等静态骨架仍在**。

这不会造成「完全不一样」，但会让页面「内容空空」。需要后端 API 可达才能看到完整数据。

## 四、微信平台固有限制（不是 bug，但会造成观感差异）

即使导入完全正确，以下差异是**小程序平台原生限制**，与设计稿会有出入：

| 设计稿特性 | 微信端实际表现 | 说明 |
|---|---|---|
| 字体 Oswald / DM Sans | 多数情况回退系统字体 | 小程序需托管字体文件 + `loadFontFace`；已注入但受网络/域名限制，失败则回退。**主要影响数字/英文，中文始终系统字体** |
| 毛玻璃 `backdrop-filter: blur()` | 被忽略，降级为半透明平卡 | 微信小程序不支持背景模糊，卡片变为不透明/半透明纯色卡 |
| 按压反馈 `:active { scale }` | 无反馈 | 微信不支持 `:active` 伪类，需改用 `hover-class` |
| 数字排版差异 | 字体回退导致字重/字间距不同 | 同上，受字体加载影响 |

这些叠加起来，可能让你主观觉得「差很多」，但**布局结构、配色、导航与设计稿是一致的**。

## 五、3 步自检，定位你的情况

请按以下顺序核对，并在任一步告诉我结果，我可进一步定位：

1. **项目目录**：微信工具左上角「项目目录」是否等于 `client\dist\build\mp-weixin`？
2. **视觉骨架**：页面是否显示**深色背景 + 底部 3 个 Tab（名片/战绩/排行）**？
   - 若是 → 导入正确，差异来自「平台限制」或「后端未运行」（见三四节）
   - 若否（白底/无 Tab/默认样式）→ 导入目录错误（见原因 1）
3. **数据**：页面标题、Tab、背景都在，但内容区空白 → 后端 API 未运行（见原因 3）

## 六、如果确认导入正确后仍想进一步逼近设计稿

可继续做的（均非必须，属渐进增强）：
- 给关键按钮加 `hover-class` 实现按压态（替代无效的 `:active`）
- 将品牌字体字体文件托管到 CDN 并在 `loadFontFace` 使用，提升数字/英文还原度
- 毛玻璃区域改为「实色半透明 + 边框高光」的等效视觉，规避微信不支持 blur 的限制

> 以上修改不改变结构正确性，仅用于消弭平台差异。是否需要我执行，请告知。
